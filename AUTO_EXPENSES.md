# Авторасходы (планировщик)

Правила авторасходов создаются в веб-интерфейсе: название, категория, **день месяца** списания, сумма. Если в месяце меньше дней (например, 31-е в феврале), используется **последний день месяца**.

Раз в сутки, когда наступает нужный день, в раздел «Расходы» добавляется запись с этой суммой и категорией. Комментарий к записи формируется как `Авто: {название}. {ваш комментарий}` (если комментарий к правилу пустой — только первая часть).

Чтобы **не создавать дубликаты** в одном месяце, у каждого правила хранится месяц последнего срабатывания (`last_applied_month`).

## Расписание Laravel

В `app/Console/Kernel.php` (метод `schedule()`) зарегистрировано:

- команда `php artisan auto-expenses:apply`
- **каждый день в 06:00** по часовому поясу приложения (`APP_TIMEZONE` в `.env`, см. `config/app.php`) — время меняется в `dailyAt('6:00')` в `app/Console/Kernel.php`, это **не** поле «день списания» в форме.

**Важно:** в правиле авторасхода «день списания» — это **число месяца** (1–31), а не время суток. Совпадение проверяется в день запуска команды (после того как `schedule:run` вызвал `auto-expenses:apply` в нужный час).

## Cron на сервере

Планировщик Laravel сам по себе не запускается. В crontab нужна строка, которая **каждую минуту** вызывает обработчик расписания.

Вариант **без `cd`** — полный путь к `artisan` (Laravel сам вычисляет корень приложения от расположения `artisan`):

```cron
* * * * * /usr/bin/php /home/user/web/family.ameliq.ru/public_html/artisan schedule:run >> /dev/null 2>&1
```

Если что-то не находит файлы (редко), попробуйте с переходом в каталог:

```cron
* * * * * cd /home/user/web/family.ameliq.ru/public_html && /usr/bin/php artisan schedule:run >> /dev/null 2>&1
```

## Ручной запуск

Проверить или применить правила вне расписания:

```bash
php artisan auto-expenses:apply
```

Посмотреть список задач и время:

```bash
php artisan schedule:list
```

## Проверка, что cron вызывает `schedule:run`

Команда **`php artisan cron:test`** пишет в `storage/logs/laravel.log` строку с временем срабатывания.

Ручной запуск (без cron):

```bash
php artisan cron:test
```

Чтобы задача шла **раз в минуту** через планировщик (как только в crontab настроен `schedule:run`), в `.env` временно включите:

```env
CRON_TEST_LOG=true
```

После проверки выключите (`false` или удалите строку), чтобы не засорять лог.

### «No scheduled commands are ready to run»

Раньше у `cron:test` в расписании было условие `when(CRON_TEST_LOG)`. Пока флаг выключен и до **06:00** не дошло, Laravel мог считать, что **нечего запускать** (фильтр отсекал единственную минутную задачу, а авторасходы — только раз в сутки в 06:00).

Сейчас `cron:test` в планировщике вызывается **каждую минуту**; запись в лог идёт только при `CRON_TEST_LOG=true`. Ручной `php artisan schedule:run` должен отрабатывать без этого сообщения (будет запускаться `cron:test`).

Команда **`auto-expenses:apply`** по-прежнему выполняется **только в 06:00** (по `APP_TIMEZONE`). В другое время её не будет в выводе `schedule:run` — это нормально. Проверить вручную: `php artisan auto-expenses:apply`.
