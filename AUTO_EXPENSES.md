# Авторасходы (планировщик)

Правила авторасходов создаются в веб-интерфейсе: название, категория, **день месяца** списания, сумма. Если в месяце меньше дней (например, 31-е в феврале), используется **последний день месяца**.

Раз в сутки, когда наступает нужный день, в раздел «Расходы» добавляется запись с этой суммой и категорией. Комментарий к записи формируется как `Авто: {название}. {ваш комментарий}` (если комментарий к правилу пустой — только первая часть).

Чтобы **не создавать дубликаты** в одном месяце, у каждого правила хранится месяц последнего срабатывания (`last_applied_month`).

## Расписание Laravel

В `bootstrap/app.php` зарегистрировано:

- команда `php artisan auto-expenses:apply`
- **каждый день в 06:00** по часовому поясу приложения (`APP_TIMEZONE` в `.env`, см. `config/app.php`)

## Cron на сервере

Планировщик Laravel сам по себе не запускается. В crontab нужна строка, которая **каждую минуту** вызывает обработчик расписания, например:

```cron
* * * * * cd /path/to/project && php artisan schedule:run >> /dev/null 2>&1
```

Подставьте реальный путь к проекту на сервере.

## Ручной запуск

Проверить или применить правила вне расписания:

```bash
php artisan auto-expenses:apply
```

Посмотреть список задач и время:

```bash
php artisan schedule:list
```

## Проверка, что cron вызывает `schedule:run`

Команда **`php artisan cron:test`** пишет в `storage/logs/laravel.log` строку с временем срабатывания.

Ручной запуск (без cron):

```bash
php artisan cron:test
```

Чтобы задача шла **раз в минуту** через планировщик (как только в crontab настроен `schedule:run`), в `.env` временно включите:

```env
CRON_TEST_LOG=true
```

После проверки выключите (`false` или удалите строку), чтобы не засорять лог.
